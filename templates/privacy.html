Student Data Privacy

How we handle your data, what we collect, and what we don't.

Last updated: February 2026

The short version

✓

We don't sell your data to anyone. Not advertisers, not data brokers, not other companies.

✓

We only collect what we need to make the platform work: your username, a hashed password, and your course progress.

✓

Your code runs in your browser. We don't see it, store it, or send it anywhere.

✓

No third-party trackers. No analytics scripts watching what you do. No ads.

✓

You can request deletion of your account and all associated data at any time.

What we collect

Account information

When you sign up, we store:

  • Username - the name you choose when creating your account
  • Password - stored as a one-way hash (SHA-256). We never store your actual password and cannot recover it.
  • Account creation date

Learning progress

As you use the platform, we track:

  • Which courses you've started
  • Number of exercises completed
  • Your streak (consecutive days of activity)
  • Course completion percentages

This data powers your dashboard so you can see your own progress. We don't use it for any other purpose.

Project idea submissions

If you submit a project idea through the Projects page, we store the idea text, your name (if provided), and skill level. This is only used to review suggestions for new tutorials.

What we don't collect

  • Email addresses - we don't ask for one
  • Real names - your username can be anything
  • Location data - we don't track where you are
  • Device fingerprints - we don't identify your device
  • Your code - all exercises run locally in your browser and are never sent to our servers
  • Browsing behavior - no page view tracking, no click tracking, no heatmaps

How your code stays private

When you use the Block Coding editor, Workflow Builder, or Git Visualizer, everything runs directly in your browser. The code you write is processed on your machine, not our servers. We have no way to see what you're building or experimenting with.

The only exception is the code execution API for the Block Coding section, which runs submitted code in a sandboxed environment to show you the output. This code is processed in memory, never saved to disk, and discarded immediately after execution.

Third-party services

We use a minimal set of external services:

  • Vercel - hosts the website. Vercel's infrastructure handles incoming web requests. See Vercel's privacy policy.
  • Upstash Redis - stores account data (usernames, hashed passwords, progress). Data is encrypted at rest. See Upstash's privacy policy.
  • Google Fonts - loads the Inter and JetBrains Mono fonts. Google may log font requests. See Google's privacy policy.

We do not use Google Analytics, Facebook Pixel, Mixpanel, Hotjar, or any other analytics or advertising service.

Cookies

We use a single session cookie to keep you logged in. It contains a session identifier - not your username, password, or any personal information. It expires when you log out or close your browser. We don't use cookies for tracking or advertising.

Data for students under 13 (COPPA)

We do not knowingly collect personal information from children under 13. Since we don't require email addresses, real names, or other personally identifiable information to create an account, the data we hold is minimal. If a parent or guardian believes their child has provided personal information, contact us and we will delete it.

Data for students in the EU (GDPR)

If you're in the EU, you have the right to:

  • Access the data we have about you
  • Correct any inaccurate data
  • Request deletion of your data
  • Export your data in a readable format

Contact us to exercise any of these rights.

School and classroom use

If you're a teacher using TrustAI in a classroom setting:

  • Student accounts only require a username and password - no email or real name needed
  • Students can use pseudonyms
  • We don't build advertising profiles on students
  • We don't share student data with third parties for non-educational purposes
  • We can delete all student accounts for a class on request

Data security

Passwords are hashed with SHA-256 before storage - we never store plaintext passwords. Data in our database is encrypted at rest. All connections to the site use HTTPS encryption. We don't store more data than we need, which limits what could be exposed in the unlikely event of a breach.

Data deletion

You can request complete deletion of your account and all associated data by contacting us. We will remove your username, hashed password, progress data, and any project idea submissions tied to your account.

Questions? If you have any questions about how we handle your data, reach out to us and we'll be happy to explain.

This policy may be updated from time to time. We'll note the date of the last update at the top of the page.
